# Documentation Policy

Documentation in `docs/` is part of the deliverable for user-visible behavior.

## Required For Future Changes

When a change introduces or alters user-visible behavior, update the relevant
documentation in the same change. This includes:
- CLI usage changes
- Configuration behavior
- Operating mode behavior
- Security or prerequisite guidance

## Review Checklist

Before submitting a change:
- Confirm README links to relevant docs pages
- Confirm docs examples match current CLI behavior
- Confirm tailnet-default and funnel-opt-in behavior is documented where relevant
